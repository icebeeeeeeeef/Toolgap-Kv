# Gate A0.1 执行结论：完整 block 覆盖停止

> 结论状态：`experimentally validated`（单一固定 testbed 上的负向适用性结论）
>
> 主结论 artifact：本地未纳入 Git 的
> `raw/a0.1/a01-20260721T190035Z-span-v2/`；逐文件 SHA-256 见本文。

## 1. 已回答的问题

在固定 vLLM、模型、模板、parser 和单工具 fixture 下，R0 实际生成的 tool-call
token 与 R1 canonical history 是否既逐 ID 一致，又能通过 APC 的完整 block 边界覆盖
assistant tool-call semantic span 的末端？

答案是：**semantic token 相同，但完整 block 覆盖不足；A0.1 为
`serialization_stop`。**

这不是性能结果、APC hit/miss 结果、CPU offload 结果，也不是 ToolGapController 的
实现证据。

## 3. 可观测结果

五个新 engine 的 R0 preflight 均从 `RequestOutput` 直接采集；五条完整 R0 token-ID
数组逐 ID 一致，且每一条 `local_prompt_equal=true`。第 5 个干净 engine 随后完成 R1。

| 观测 | 值 | 含义 |
| --- | ---: | --- |
| R0/R1 assistant span | `[178, 198)` 两侧相同 | span 由完整模板前缀推导的 assistant-message 区域内 marker 得出 |
| `semantic_span_equal` | `true` | tool-call wire envelope 的 token 没有漂移 |
| LCP | `199` | R0/R1 首 199 个 token 完全相同 |
| vLLM block size | `16` | 运行时读取，不是预设值 |
| `reusable_full_block_ceiling` | `192` | APC 可复用完整 block 的上限 |
| `a_end_mod_block_size` | `6` | semantic span 尾端不在完整可复用 block 内 |
| `coverage_slack` | `-6` | `192 - 198`，未覆盖 semantic end |
| mismatch region | `after_assistant_semantic_or_r0_exhausted` | 首个不一致不在 assistant semantic span 内 |

因此，最后 6 个 semantic token 所在的 block 不能因 R1 在第 199 token 后分歧而成为
完整 APC prefix block。A0.1 的预注册规则要求
`reusable_full_block_ceiling >= a_end`；该条件不成立，故停止。

## 4. 判决与边界

### 已证实

- 在该 pin 与 fixture 下，模型实际生成的 tool-call semantic token 可 round-trip 到 R1；
- 该 semantic span **不能完整落在** APC 可复用的 full block 前缀内；
- A0.2 的输入前提不成立，A0.2、ToolGapController、custom offload wrapper 与 forced-path
  runner 均不启动。

### 未证实，且不得推断

- 任何 GPU APC hit/miss、CPU restore、HBM 压力、队列、TTFT、吞吐或性能收益；
- 其它模型、模板、tool schema、多轮工具调用或生产 agent 负载；
- “所有 vLLM 工具调用都不可复用”或“存在 semantic token serialization 漂移”。

## 5. Harness 处置记录

最初 run `a01-20260721T183230Z` 在 R0 已稳定后得到 `invalid_run`：旧 adapter 错把
Hermes 模板协议说明中的 `<tool_call>` marker 当成全局唯一性失败。它没有用作结论。

修复 commit `8d697da` 将 marker 搜索范围限制为经两段完整模板前缀验证得到的 candidate
assistant-message 区域，并新增回归测试。随后一次 scoped-span 运行与最终 v2 运行得到相同
数值；最终 run 以 commit `a576325` 将该观察算法显式版本化为 v2。该版本更新不改变 fixture、
模型、vLLM、判决阈值或结果，只修正 adapter provenance。
