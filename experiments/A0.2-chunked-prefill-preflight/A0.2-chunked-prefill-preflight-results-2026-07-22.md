# A0.2 chunked-prefill preflight：结果记录（2026-07-22）

> 证据状态：`experimentally validated`，但仅限于下面定义的 scheduler-pin admission
> 命题；不构成 A0.2 压力实验、CPU offload、性能或 ToolGap runtime 的结果。

## 1. 目的与问题

A0.1R 曾在 `enable_chunked_prefill=False` 下完成 3/3 no-pressure APC admission，但 pinned
vLLM 明确警告该模型不官方支持关闭 chunked prefill。本实验不复跑 A0.1R，也不改其历史
bundle；它只验证受支持的 `enable_chunked_prefill=True` 是否仍使同一 canonical R0/R1 pair
在公开 `RequestOutput.num_cached_tokens` 中 materialize 已冻结的完整前缀 `C=192`。

这次验证对应 [配置审计](../0001-mechanism-feasibility/A0.2-chunked-prefill-configuration-audit.md)
的三 ordinal preflight，而非 A0.2 comparative matrix。

| ordinal | R0 cached | R1 cached | LCP | semantic span equal | verdict |

所有 ID-sequence hash 固定使用 A0.1R prefix anchor 的同一公式：
`sha256(json.dumps(ids, separators=(",", ":")).encode("utf-8"))`，即紧凑 JSON integer list、
UTF-8、**无末尾换行**。三个 R0 ID 序列的 SHA-256 都是
`5bd787e46e48a35f372cba688ce4cdb9c3eb0e289c76ec0b178c14bd46eb932e`；三个 R1 prompt-ID
序列的 SHA-256 都是
`c3c9f07fce5ecd25eaf82f2b67e5e576ef0bb08fd8ceca8ab48f2856071283e8`。可用
`jq -cj '.r0_ids' raw/preflight/preflight-o01-a01/r0.json | shasum -a 256` 或将 `.r0_ids`
替换为 `.prompt_token_ids` 复算。

初稿误写的是 `jq -c` 输出（含一个末尾 LF）的 hash；二者均来自相同 raw IDs，但前者的
换行口径未在报告中声明，故不再保留。三个 `tool_call_id` 不同，因而 R1 JSON bundle 不字节
相同；这表明 runner 没有把静态 ID 偷写进 fixture。

## 5. 可以与不可以得出的结论

可以得出：在以上 pin、固定 canonical pair、单请求无压力环境中，受支持的
chunked-prefill scheduler 仍把 192 个已登记的完整前缀 token 通过公开 request accounting
materialize 出来。A0.1R 的 `False` 配置不再是 A0.2 scheduler pin 的必要前提。

不可以得出：

- 压力下 chunked-prefill interleaving 的 per-request accounting 仍正确；
- A0.2 是否存在可归因、有经济价值的 miss；
- CPU restore、queue/TTFT、吞吐或任何性能收益；
- ToolGap lifecycle runtime、retain/offload policy 的价值或正确性。

因此 A0.2 若继续，仍必须在每个 comparative run 中做 anchor/accounting check。无压力
`R1.cached=192` 是本 preflight 已验证的 baseline；压力后 block-aligned 的
`0 <= R1.cached < 192` 是待归因的候选 miss，不是 contract change。缺失、非 block-aligned、
超过 192 或超过实际 prompt 长度的计数才是 `accounting_contract_change`，不得被改解释为
hit 或 miss。

## 6. 下一步

本 gate 已移除“unsupported `enable_chunked_prefill=False` 配置”这一阻塞项。下一件合理的
工作是：基于 `chunked_prefill=True` 修订并评审 A0.2 stock-sufficiency spec 的运行 pin 与
per-run accounting guard，然后才实现/运行容量压力下的 S0/S1 matrix。该步骤依旧不实现
ToolGap runtime。
