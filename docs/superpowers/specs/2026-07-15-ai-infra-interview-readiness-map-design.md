# AI Infra 面试就绪知识地图设计

> 状态：已确认的组织基线
>
> 日期：2026-07-15
>
> 目标岗位：2027 校招 LLM Serving / 大模型推理平台 / AI Infra 平台研发 / 推理调度 / 模型服务后端

## 1. 目标

在完成并能够深挖 ToolGap-KV 的同时，建立一份可验收的面试知识清单，覆盖：

- 基础知识口述与连续追问；
- 真实系统源码、数据流、控制流与设计取舍；
- 非 LeetCode 领域手撕与本地测试；
- LeetCode 类算法题；
- 可复现实验、性能分析与失败边界；
- ToolGap-KV 的真实实现证据与项目深挖。

最终产物是一个浏览器可直接打开的 HTML 思维导图。节点可以展开或收起，并展示目标熟练度、验收标准、项目关联、优先级和学习状态。

## 2. 组织原则

采用“主题树 + 能力维度”，不使用单一的笼统熟练度。

主题主干暂定为：

1. 模型与推理原理
2. PyTorch 与张量编程
3. LLM 推理引擎
4. KV Cache 与内存系统
5. 调度、吞吐与服务系统
6. GPU 与硬件基础
7. 分布式推理与通信
8. 系统与语言基础
9. 性能分析与实验方法
10. 领域手撕
11. LeetCode 算法
12. ToolGap-KV 项目深挖

主题之间允许交叉引用，但每个知识点只有一个主要归属，避免重复统计学习进度。

## 3. 熟练度模型

每个节点分别评估四个能力维度：

- **原理**：能否从第一性原理解释机制、复杂度和资源约束，并承受连续追问。
- **源码**：能否定位真实系统实现，解释核心对象、数据流、控制流、并发语义和 trade-off。
- **手写**：能否在限定时间内完成简化实现、说明删减边界并通过本地测试。
- **实验**：能否设计并运行复现实验，正确解释指标、归因、噪声、负面结果和适用边界。

每个维度使用四级目标：

| 等级 | 可验收定义 |
|---|---|
| L0 了解 | 知道用途和术语；不视为面试准备完成 |
| L1 能讲 | 脱稿讲清基本原理、输入输出、主要复杂度和典型用途 |
| L2 能深挖 | 能解释源码路径、关键数据结构、设计取舍、失败模式和替代方案 |
| L3 能实现 | 能独立手写或修改实现，完成调试、本地测试或可复现实验 |

L3 不要求所有主题都“从零重写生产系统”。每个主题必须单独定义实现范围和时间限制。

## 4. 节点记录模板

每个知识节点至少记录：

```text
主题名称
岗位权重：P0 / P1 / P2 / P3
目标熟练度：原理 L? / 源码 L? / 手写 L? / 实验 L?
为什么需要掌握
知识边界
必须回答的问题
必须完成的手写或实验
源码阅读锚点
常见错误与追问
与 ToolGap-KV 的关系
完成证据
当前状态
```

优先级定义：

- **P0**：不掌握就不应开始目标岗位面试。
- **P1**：高频或能够明显提高通过率，应在集中投递前完成。
- **P2**：岗位相关但依赖具体面试官或 JD，可在投递期间补齐。
- **P3**：超出目标岗位主线，作为防守性知识或未来扩展。

## 5. 单主题讨论流程

每个主题按以下顺序讨论：

1. 说明它对目标岗位和 ToolGap-KV 的真实价值；
2. 划定需要掌握和明确不需要掌握的边界；
3. 拆出不过细但可验收的二级知识节点；
4. 为每个节点设置四维目标等级；
5. 给出面试追问、手写题和实验任务；
6. 用对抗式审查检查是否过浅、过宽或与岗位错位；
7. 用户确认后写入持续维护的知识地图 Markdown。

讨论顺序不等于最终思维导图的目录顺序。优先讨论能够支撑项目实施、又是面试高频门槛的主题。

## 6. ToolGap-KV 边界

ToolGap-KV 当前材料中的 `roadmap`、`shipped`、`experimentally validated` 和 `simulated` 必须保持原义。

知识地图不得因为存在项目设计文档，就把下列能力视为已经掌握：

- 真实 vLLM 接入和源码修改；
- 候选人自有生命周期控制器；
- GPU 性能测量；
- 故障注入、清理和恢复证据；
- 定量 break-even 或尾延迟结论。

项目完成证据与个人学习证据分别记录，然后在面试深挖节点中建立关联。

## 7. 输出文件

计划维护两个产物：

1. Markdown：讨论期间的权威内容源，便于审阅和版本管理；
2. HTML：从已确认 Markdown 汇总生成的最终交互式思维导图。

HTML 至少支持：

- 节点展开与收起；
- 关键词搜索；
- 按优先级、目标熟练度和完成状态筛选；
- 节点详情展示；
- ToolGap-KV 关联节点和独立面试准备节点的视觉区分；
- 本地直接打开，不依赖在线服务。

## 8. 完成条件

只有同时满足以下条件，才称为“可以开始面试”：

- 所有 P0 节点达到各自目标等级并留下验收证据；
- P0 领域手撕和算法题能够在限制时间内通过本地测试；
- ToolGap-KV 的简历表述只使用已有证据支持的 claim state；
- 至少完成一次覆盖基础知识、领域手撕、算法题和项目深挖的综合模拟面试；
- 对未完成的 P1/P2/P3 节点能够明确说明风险，而不是把它们误判为已掌握。
