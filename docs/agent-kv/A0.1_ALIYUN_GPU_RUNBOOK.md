# A0.1 阿里云 GPU 租用与开实验导引

> 状态：`roadmap`。本指南只把 A0.1 harness 带到可真实执行的 testbed；租到机器、驱动可用或脚本启动都不等于 A0.1 已通过。

## 0. 这次只租什么

目标是运行一次单卡、单进程的 A0.1 token round-trip experiment，不是训练、部署线上服务或搭建集群。

选择：**一台按量付费、非抢占、整卡 GPU ECS**，一张显存至少 24 GiB 的 NVIDIA 计算卡，Linux，100 GiB 系统盘，公网 IPv4 只用于 SSH 和下载模型。

首选规格是 `ecs.gn7i-c8g1.2xlarge`（A10 × 1、24 GiB）；若购买页在目标可用区无货，选同系列 `ecs.gn7i-c16g1.4xlarge`（仍为 A10 × 1、24 GiB）。不要为了“更便宜”改为 vGPU、抢占式或多卡：前两者引入驱动/回收变量，后者改变 A0.1 的单卡证据边界。24 GiB 是 Qwen2.5-7B 单请求校验的工程下限，不是性能承诺；购买页实际库存和价格才是事实。[GPU 规格族](https://help.aliyun.com/zh/ecs/user-guide/gpu-accelerated-compute-optimized-and-vgpu-accelerated-instance-families-1)

## 1. 下单前的三项检查

- [ ] 已登录阿里云主账号；若购买中国内地地域，已完成实名认证。
- [ ] 账户没有欠费，有足够的按量付费额度/余额。GPU 配额按**地域、规格族、付费方式**分别计算；先到「配额中心 → 云服务器 ECS 规格配额」选择候选地域，确认对应 GPU 规格的 `Used < Quota`。若为 0 或创建时提示配额不足，申请提升后再下单。[ECS 配额管理](https://help.aliyun.com/zh/doc-detail/184116.html)
- [ ] 打开 ECS「自定义购买」页，在至少两个相近地域/可用区查看 `gn7i` 单卡规格是否有货。地域创建后不能直接迁移；优先选择你网络访问顺畅、且能买到实例的地域。

不要先买包年包月，也不要用抢占式。抢占实例可能因库存或市场价格被自动回收；A0.1 的五次 fresh-engine preflight 被中断后没有可解释的实验结果。[创建 GPU 实例](https://help.aliyun.com/zh/egs/user-guide/create-a-gpu-instance/)

## 2. 控制台逐项下单

进入「ECS 控制台 → 实例 → 创建实例 → 自定义购买」，按下面填写。控制台文案可能小幅变化，以字段含义为准。

| 区域 | 选择 | 原因 |
| --- | --- | --- |
| 付费类型 | **按量付费** | 可按秒结算并在实验后释放。 |
| 地域 / 可用区 | 有 `gn7i` 单卡库存的一个可用区 | A0.1 不需要跨区资源。 |
| 实例 | `ecs.gn7i-c8g1.2xlarge`；无货时 `ecs.gn7i-c16g1.4xlarge` | 一张完整 A10 24 GiB；不要改成 vGPU/多卡。 |
| 镜像 | 优先 **Alibaba Cloud Linux 3/4 预装 NVIDIA GPU 驱动和 CUDA 的公共镜像** | 降低主机驱动安装风险；该镜像支持 `gn7i`。 |
| 网络 | VPC；创建默认 VPC/交换机即可 | 单机实验没有额外网络拓扑。 |
| 公网 | 分配固定公网 IPv4；带宽按流量计费即可 | 只需 SSH、Git 与 Hugging Face 下载。 |
| 系统盘 | 高效云盘或 ESSD，**100 GiB**，勾选“随实例释放” | 放模型缓存、vLLM 源码、venv 与 raw artifact；不用数据盘。 |
| 安全组 | 新建普通安全组 | 后续仅放行自己公网 IP 的 SSH。 |
| 登录凭据 | **密钥对**、用户名 `ecs-user` | 官方建议普通用户而非 `root`。 |
| 自动释放 | 设置为“预计准备 + 一次实验 + 1 小时缓冲”后的绝对时间 | 是本次最重要的费用兜底。首次建议 8 小时，若模型下载较慢再延长。 |

在镜像区域若看到“自动安装 GPU 驱动 / Tesla 驱动”选项，保持启用并选择与该镜像兼容的 CUDA 版本。公共 GPU 镜像/自动安装会处理 Tesla driver、CUDA 与 cuDNN；驱动安装通常还需要 10–20 分钟，期间不要安装其他 GPU 软件。[预装驱动镜像](https://help.aliyun.com/zh/ecs/user-guide/alibaba-cloud-linux-with-pre-installed-nvidia-gpu-drivers)；[自动安装 Tesla driver](https://help.aliyun.com/zh/egs/user-guide/automatically-install-or-load-tesla-drivers-via-imgaes)

安全组的入方向只保留一条：`TCP / 22 / 你的当前公网 IPv4/32`。不要开放 `0.0.0.0/0`，也**不要开放 8000**：A0.1 使用进程内 `LLM.chat()`，不启动 HTTP 服务。创建后若控制台快捷规则放开了 SSH 全网，立即删掉或收窄它。[安全组最小权限配置](https://help.aliyun.com/zh/ecs/user-guide/start-using-security-groups)

核对订单的实例、磁盘、带宽与自动释放时间后再付款。按量付费实例适合临时测试；抢占式虽然便宜，但存在回收机制。[ECS 计费方式](https://help.aliyun.com/zh/ecs/instance-types)

## 3. 租用完成验收：先不装项目

实例状态变为「运行中」后，先在控制台确认：有公网 IPv4、自动释放时间可见、系统盘 100 GiB、实例是一张完整 GPU。

在本地终端执行（将路径和 IP 替换为自己的）：

```bash
chmod 600 ~/Downloads/toolgap-kv.pem
ssh -i ~/Downloads/toolgap-kv.pem ecs-user@<PUBLIC_IP>
```

第一次连接要核对 SSH host fingerprint；进入服务器后：

```bash
nvidia-smi
df -h /
python3 --version
```

验收标准：`nvidia-smi` 能看到一张 A10（或你下单的完整 GPU）和 driver；根分区可用空间接近 100 GiB；Python 为 3.10 及以上。若驱动仍在安装，先等待自动安装日志显示成功；若 `nvidia-smi` 失败，不开始安装 vLLM，先解决镜像/driver 问题。

密钥登录与限制 SSH 来源的官方步骤见：[使用 OpenSSH 登录 Linux ECS](https://help.aliyun.com/zh/ecs/user-guide/connect-to-a-linux-instance-by-using-an-ssh-key-pair)。也可先用控制台 Workbench 排障，但长命令应在 SSH/tmux 中执行。

## 4. 让服务器具备 A0.1 启动条件

### 4.1 先解决源码来源

当前本地 A0.1 runner 在代码评审与提交前不能假设已经存在远端仓库。因此二选一：

1. **推荐**：先在本地完成 A0.1 变更提交并 push；服务器上 clone 对应分支。
2. 临时调试：在本地使用 `rsync` 仅同步当前 checkout 到服务器；不要把 SSH 私钥、`.git` 中的凭据或本地环境文件同步过去。

以下命令假定代码已在远端分支中：

```bash
sudo dnf install -y git || sudo yum install -y git
git clone https://github.com/icebeeeeeeeef/Toolgap-Kv.git ~/toolgap-kv
cd ~/toolgap-kv
git checkout <包含 A0.1 runner 的 commit>
```

### 4.2 建立项目要求的 Python 与 vLLM pin

项目要求 Python ≥3.10。若第 3 节显示的 Python 较旧，先换用带 Python 3.10+ 的镜像或在该机上安装一个独立 Python；不要在系统 Python 上硬改包。

```bash
cd ~/toolgap-kv
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e .

git clone https://github.com/vllm-project/vllm.git ~/src/vllm
cd ~/src/vllm
git checkout 752a3a504485790a2e8491cacbb35c137339ad34
python -m pip install -e .

python - <<'PY'
import vllm
print(vllm.__version__)
assert vllm.__version__ == "0.25.1"
PY
```

不要直接采用阿里云当前官方 `vllm` 容器镜像作为 A0.1 的项目环境：它解决 host/容器 GPU 接入，但其文档列出的版本并不等于本项目所需的 `v0.25.1` pin。若 `pip install -e ~/src/vllm` 构建失败，保留错误日志并停止；先解决 pin 的依赖/编译兼容性，不能换成另一个 vLLM 版本后继续声称是同一实验。[阿里云 vLLM 容器说明](https://help.aliyun.com/zh/egs/use-cases/use-a-vllm-container-image-to-run-inference-tasks-on-a-gpu-accelerated-instance)

### 4.3 记录模型 immutable revision 并启动

`Qwen/Qwen2.5-7B-Instruct` 的模型和 tokenizer 必须使用同一 Hub commit。先解析 SHA：

```bash
source ~/toolgap-kv/.venv/bin/activate
python -m pip install huggingface_hub
MODEL_SHA="$(python - <<'PY'
from huggingface_hub import HfApi
print(HfApi().model_info("Qwen/Qwen2.5-7B-Instruct", revision="main").sha)
PY
)"
printf '%s\n' "$MODEL_SHA"
```

确认输出是 40 位 SHA 后，执行 A0.1。选择 `disabled` 只是本次固定、可记录的 chunked-prefill 配置，不是性能调优：

```bash
cd ~/toolgap-kv
source .venv/bin/activate
RUN_ID="a01-$(date -u +%Y%m%dT%H%M%SZ)"
PYTHONPATH=src python scripts/run_a01.py \
  --model-revision "$MODEL_SHA" \
  --tokenizer-revision "$MODEL_SHA" \
  --chunked-prefill disabled \
  --run-id "$RUN_ID"
```

这才是“可以开启实验”的最小验收点。脚本会自行执行五次 fresh-engine R0 preflight；退出码 0 对应 `pass`，非 0 也可能是应保留的 `invalid_run` 或 `environment_blocked`，不是可以删除后重跑的普通失败。

## 5. 实验结束：先带走证据，再释放

无论结果是 `pass`、`serialization_stop`、`invalid_run` 还是 `environment_blocked`，都保留 run 目录：

```bash
cd ~/toolgap-kv
RUN_ID=<刚才命令打印出的 run-id>
tar -czf "$RUN_ID.tar.gz" "experiments/0001-mechanism-feasibility/raw/a0.1/$RUN_ID"
exit
```

回到本地终端后，重新填写同一个 run ID：

```bash
RUN_ID=<刚才命令打印出的 run-id>
scp -i ~/Downloads/toolgap-kv.pem \
  ecs-user@<PUBLIC_IP>:~/toolgap-kv/"$RUN_ID".tar.gz .
```

确认本地压缩包可解开后，到「ECS 控制台 → 实例 → 实例状态 → 释放」选择立即释放，并检查系统盘是否随实例释放；随后在账单中检查是否遗留 EIP、独立云盘或快照。**普通停止可能仍继续计费**；节省停机可暂停 GPU/计算费用，但会保留云盘等资源，且下次因库存不足可能无法重启，因此不适合作为这次短实验的收尾动作。[按量付费与停止计费规则](https://help.aliyun.com/zh/ecs/pay-as-you-go-1)；[释放实例与残留资源](https://help.aliyun.com/zh/doc-detail/163834.html)

## 6. 两个常见卡点

| 现象 | 不要做什么 | 正确动作 |
| --- | --- | --- |
| 购买页没有 `gn7i` 或下单提示配额/库存不足 | 不要改买 vGPU 或多卡 | 换可用区/地域；检查对应地域的按量 GPU 配额；必要时申请配额。 |
| `nvidia-smi` 不可用 | 不要立刻装 vLLM 或混装多个 CUDA driver | 等待镜像自动安装完成；查看安装日志；不通则重建实例并选择支持自动 Tesla driver 的镜像/规格。 |
| vLLM pin 无法安装 | 不要静默换成最新版 vLLM | 保存构建日志；先确认 Python、driver/CUDA 和 pin 的兼容性，再决定修复路径。 |
| SSH 连不上 | 不要放开 `0.0.0.0/0:22` | 检查公网 IP、实例运行状态和安全组是否只允许当前公网 IP；必要时用 Workbench/VNC 排障。 |

本 runbook 的终点是：你拿到一个可 SSH 的单卡 GPU testbed，驱动已验证，自动释放已设置，A0.1 的固定 vLLM/model pin 已准备好，并能执行 `run_a01.py`。它不替代 A0.1 结果评审。
